<p>Este dashboard utiliza distintos paquetes para la visualización de gráficos interactivos: highcharter, plotly y ggplot; De igual manera se apoya en el universo de paquetes tidyverse. A continuación se detallan los distintos modulos.</p>
<h2 style="color: #2e6c80;">Motor Portfolio TPL:</h2>
<p>Este modulo muestra el performance de una cartera de Responsabilidad Civil Vehiculos.</p>
<p>Despliega gráficos descriptivos con la opción de cambiar la variable de estudio y muestra indicadores en base a los inputs seleccionados. Realiza una simulación de frecuencia y severidad de siniestros y despliega gráficos de barras condicionados a inputs de retención de Reaseguro. Muestra también una tabla con el detalle por póliza.</p>
<h2 style="color: #2e6c80;">Long-Term IRate Forecast:</h2>
<p>Forecast de la tasa de interes a largo plazo del país seleccionado, utilizando<br />técnicas de series de tiempo en data mensual. Todos los modelos son desarrollados utilizando el paquete forecast.</p>
<p>En el panel Overview se observa un Mapa de calor con las tasas agrupadas por Cuatrimestre de cada país disponible. Se despliega también gráficos de descomposicion de serie de tiempo STL -Seasonal and Trend decomposition using Loess- asi como gráficos de Autocorrelación y Autocorrelación Parcial.</p>
<p>Se modela la serie de tiempo en data de entrenamiento utilizando varias técnicas y se realiza el forecast en la data de prueba, mostrando un cuadro comparativo de los modelos y pronosticando el seleccionado por el usuario.</p>
<h2  style="color: #2e6c80;">IBNR Development Triangles:</h2>
<p>Forecast de reclamos incurridos no reportados (IBNR) por medio de triangulos actuariales, utilizando distintas técnicas:  Chain Ladder, Mack Chain Ladder y Bootstrap Chain Ladder. Para cada técnica se muestra gráficos de diagnóstico y los resultados arrojados. En el panel Overview se observa un Mapa de. Todos los modelos son desarrollados utilizando el paquete ChainLadder.</p>
<h2 style="color: #2e6c80;">Emergency Monitor:</h2>
<p>Monitoreo del estado del clima en situaciones de emergencia. En el primer panel se visualiza el estado actual de la región seleccionada y el pronóstico en horas y dias susesivos utilizando para la recolección de data en tiempo real la API DarkSky. En el segundo panel se utiliza la API de twitter para desplegar un timeline de cuentas informativas asi como graficar la frecuencia de tweets que contienen las palabras introducidas por el usuario en los últimos 6-9 dias. Se utiliza el paquete rtweet para recolectar los datos a traves de la API de twitter y para desarrollar los gráficos.</p>
